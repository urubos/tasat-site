[2026-04-11d] — Bounded Settle Follow-up Runs

Infrastructure

  • Bounded settle follow-up runs — Removed the old settle-loop sleep from `pr-orchestrate.yml`. Each workflow run now performs a single orchestrator pass instead of waiting in a long loop. When the orchestrator reports settle states (`COPILOT_WORK_SETTLING`, `COPILOT_WORK_IN_PROGRESS`), the workflow does a short `sleep 30` and then triggers a `workflow_dispatch` follow-up run. Pushes and reviews can still naturally trigger additional runs via `pull_request:synchronize` and `pull_request_review:submitted`.

[2026-04-11c] — LLM-Optimized Analysis Formats & Roundtable Prompt

Schema — 9 LLM-Optimized Formats

  • `analysis.tension_map` — Unresolved creative tensions between persona analyses, encoded as opposing poles with generative prompts. Provokes novel synthesis rather than passive retrieval.
  • `analysis.analogical_bridges` — Structured source-target domain mappings with transfer validity scores (0-1) and limitations. Enables cross-domain reasoning.
  • `analysis.generative_seeds` — Compact symbolic encodings of analysis reasoning skeletons designed for LLM decompression. Not intended for human reading.
  • `analysis.atomic_propositions` — Self-contained factoids with retrieval tags, optimized for RAG. Each proposition can be independently embedded and retrieved.
  • `analysis.semantic_triples` — Subject-predicate-object triples forming a knowledge graph. Merge across ideas for cross-catalog queries.
  • `analysis.counterfactual_tree` — Branching what-if tree with cascading effects and real-world parallels. Expands choice_points into systematic scenario exploration.
  • `analysis.decision_context_tags` — Pre-computed relevance annotations for specific decision-making contexts (e.g., gene-drive-policy, SETI-search-strategy).
  • `analysis.scenario_template` — Parameterized scenario with substitutable variables and an invariant structural insight that holds regardless of substitution.
  • `analysis.debate_continuation` — Structured handoff with open threads, unanswered critiques, and a continuation prompt for resuming the dialectic in future sessions.

Catalog

  • `non-human-uplift-civilization` — Reference implementation of all 9 LLM-optimized fields: 3 tension maps, 3 analogical bridges, 1 generative seed, 10 atomic propositions, 12 semantic triples, 5 counterfactual tree nodes (including nested branches), 4 decision context tags, 1 scenario template with 8 variables, and debate continuation with 2 open threads and 1 unanswered critique.

Prompts

  • `roundtable.prompt.md` — New prompt for multi-persona roundtable discussions. Supports freeform and structured modes. Generates all 9 LLM-optimized formats as structured output.
  • `fulltext-analysis.prompt.md` — Added step 6b with instructions for generating all 9 LLM-optimized formats after the dialectic.

Tooling

  • `tools/extract_epub_text.py` — New utility for extracting prose text from EPUB files and extracted EPUB directories. Strips front/back matter and collapses blank lines to output clean text for full-text analysis.

[2026-04-11b] — Auto-Merge Pipeline

Infrastructure

  • Auto-merge for `main` branch — The orchestrator now auto-merges PRs targeting `main` (and `staging`) when Copilot review passes with no unresolved threads, all checks are green, and the PR is merge-clean. Controlled by `EFA_AUTO_MERGE_ENABLED=1` in `pr-orchestrate.yml`.
  • Pre-merge sanity check — Before auto-merging, the orchestrator runs `validate_catalog.py` and `generate_site.py` as a final gate, and re-verifies that all review threads are resolved via a live API check. If any check fails, it falls back to posting a "ready to merge" comment instead.

[2026-04-11a] — Orchestrator Re-review & Thread Resolution

Infrastructure

  • Fix Copilot re-review request — `request_copilot_review()` now uses the REST API with an explicit JSON body (`{"reviewers":["Copilot"]}` via `--input -`) instead of `gh pr edit --add-reviewer copilot-pull-request-reviewer` which fails because the bot is not a collaborator. After the API call, verifies the request landed by checking pending reviewers (the API returns 201 even when requests are silently dropped while Copilot is already reviewing). When verification fails, the request is recorded as unconfirmed and cooldown is not started, allowing prompt retries.
  • No auto-resolve of review threads — `address_feedback()` no longer calls `resolve_review_threads()` after copilot-cli applies fixes. Threads can become `isOutdated` after any push, even when the underlying feedback is still valid; auto-resolving here could hide still-valid Copilot feedback and prematurely lower unresolved-thread counts. Threads are now left open for explicit re-review or human confirmation. The `resolve_review_threads()` helper remains available for explicit use.
  • Unified re-review path — `address_feedback()` now uses the shared `request_copilot_review()` function for re-review requests instead of a separate inline `gh pr edit` call that lacked verification and fallback logic.

[2026-04-11] — Retire Webhook Infrastructure

Infrastructure

  • Removed webhook receiver — Deleted `tools/webhook_receiver.py`, `tools/webhook_manage.sh`, and `tools/orchestration_watchdog.py`. The GitHub Actions workflow (`pr-orchestrate.yml`) now handles the full PR review loop; the Codespace-based webhook receiver is no longer needed.
  • Removed bridge extension — Deleted `.vscode-extensions/efa-orchestrator-bridge/` and its setup in `.devcontainer/setup.sh`. The legacy feedback-file handoff path in the orchestrator has been removed; copilot-cli is the sole fix application method.
  • Simplified devcontainer — Removed port 8585 forwarding and the webhook watchdog section from `.devcontainer/start-site.sh`. The devcontainer now only starts the site preview server.
  • Cleaned up orchestrator — Removed `WEBHOOK_EVENTS_FILE`, `read_recent_webhook_events()`, and all webhook event reads from `tools/pr_review_orchestrator.py`. Copilot work-session detection and recent-activity checks now rely solely on GitHub API timeline data.
  • Simplified configuration — Removed `EFA_PR_ORCHESTRATOR_ENABLED`, `WEBHOOK_PORT`, `WEBHOOK_BIND_ALL`, and watchdog settings from `.env.example`. Legacy webhook secret filenames remain ignored in `.gitignore` to reduce the risk of accidentally committing local secret material.
  • Recommended PAT checkout for fix pushes — `pr-orchestrate.yml` prefers `COPILOT_PAT` for the checkout step when it is available, and otherwise falls back to `github.token`. Using the PAT is recommended for copilot-cli fix pushes because it allows fix commits to trigger `pull_request: synchronize` events, enabling CI re-runs and Copilot re-review.

[2026-04-10g] — Decision-Support Schema for LLM Consumers

Schema

  • `decision_contexts` — New optional field on ideas. Controlled enum of decision domains (`policy-formation`, `risk-assessment`, `technology-strategy`, `resource-allocation`, `regulation`, `infrastructure-planning`, `security-defense`, `public-health`, `economic-planning`, `environmental-management`, `ethics-governance`, `innovation-rd`). Enables LLM consumers to route ideas to relevant decision-maker contexts without semantic search.
  • `temporal_context` — New optional object on ideas. Contains `real_world_emergence` (year when the concern appeared in real-world contexts) and `urgency_trajectory` (enum: `emerging`, `growing`, `peak`, `declining`, `dormant`, `recurring`). Enables temporal reasoning about idea relevance.
  • `abstraction_level` — New optional field on ideas. Enum: `civilization`, `sector`, `organization`, `technology`, `individual`. Enables hierarchical matching across query granularities.
  • `choice_points` — New optional field on `analysis`. Array of decision points within the `mechanism_chain` where intervention could alter the outcome. Each entry specifies `mechanism_step`, `decision`, `actual_path`, and `alternative_outcomes`. Enables counterfactual "what if we intervened at step X?" reasoning for policy analysis.

Prompts & Skills

  • Full-text analysis prompt — Added `decision_contexts`, `temporal_context`, `abstraction_level` to idea identification step. Added `choice_points` documentation to analysis fields.
  • Story-idea extraction skill — Added decision-support fields to idea identification and analysis block documentation.
  • Copilot instructions — Updated schema field reference with all new fields.
